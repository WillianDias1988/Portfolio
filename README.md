# Portfolio

Portfólio profissional para apresentar minha experiência, projetos e capacidade de entrega em engenharia de software.

## Visão geral

Este repositório foi estruturado para servir como base do meu portfólio técnico. O objetivo é comunicar com clareza:

- quem eu sou como engenheiro de software;
- quais tipos de problemas eu resolvo;
- quais tecnologias utilizo com mais profundidade;
- quais decisões técnicas valorizo em produtos reais.

## Objetivos do projeto

- Consolidar uma vitrine profissional com foco técnico;
- Demonstrar capacidade de organização, documentação e evolução incremental;
- Servir como base para publicação de uma aplicação web de portfólio;
- Expor decisões de arquitetura, qualidade e experiência do desenvolvedor.

## O que este projeto demonstra

Este repositório foi organizado para evidenciar competências que normalmente são esperadas de um perfil sênior:

- estruturação de projeto com preocupação de manutenção;
- documentação clara para onboarding e colaboração;
- preparação para CI;
- padronização mínima de ambiente;
- visão de produto e roadmap técnico.

## Stack prevista

A stack final pode evoluir conforme a implementação, mas a base pensada para este portfólio considera:

- Front-end: React ou Next.js com TypeScript;
- Estilização: Tailwind CSS ou CSS Modules;
- Qualidade: ESLint, Prettier e testes automatizados;
- CI/CD: GitHub Actions;
- Deploy: Vercel, Netlify ou infraestrutura equivalente.

## Como evoluir este repositório

### Etapa 1 — Base institucional

- adicionar identidade visual;
- publicar apresentação profissional;
- incluir links de contato e redes relevantes;
- definir stack final.

### Etapa 2 — Aplicação funcional

- implementar home do portfólio;
- criar seções de projetos, experiência, habilidades e contato;
- tornar o layout responsivo;
- adicionar acessibilidade básica.

### Etapa 3 — Qualidade de engenharia

- configurar lint e formatter;
- incluir testes unitários;
- adicionar pipeline de validação automática;
- medir performance e ajustar Core Web Vitals.

### Etapa 4 — Diferenciais de portfólio

- documentar decisões técnicas;
- publicar estudos de caso dos projetos;
- incluir screenshots, GIFs e demo pública;
- registrar roadmap e melhorias.

## Boas práticas esperadas

Para que este projeto represente adequadamente um portfólio sênior, a evolução deve priorizar:
